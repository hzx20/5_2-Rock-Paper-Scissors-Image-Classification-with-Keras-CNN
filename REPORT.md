### 🔹 步骤 2：使用 Keras 构建并训练 CNN 模型

#### 2.1 图像预处理参数

| 参数 | 值 |
|------|-----|
| 输入图像尺寸 | 48 × 48 像素 |
| 像素归一化 | [0, 255] → [0, 1] |
| 数据格式 | channels_first（通道优先） |
| Batch Size | 64 |
| 训练样本（每类） | 200 张（共 600 张） |
| 测试样本（每类） | 80 张（共 240 张） |

#### 2.2 模型架构（CNN）

```
┌─────────────────────────────────────────┐
│  Input: (3, 48, 48) 三通道彩色图像     │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Conv2D(32, 3×3, ReLU, padding='same') │  ← 提取32种基础特征
└──────────────────┬──────────────────────┘
┌──────────────────▼──────────────────────┐
│  MaxPooling2D(2×2)                      │  ← 降采样，减少计算量
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Conv2D(64, 3×3, ReLU, padding='same') │  ← 提取64种复杂特征
└──────────────────┬──────────────────────┘
┌──────────────────▼──────────────────────┐
│  MaxPooling2D(2×2)                      │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Conv2D(64, 3×3, ReLU, padding='same') │  ← 更深层特征提取
└──────────────────┬──────────────────────┘
┌──────────────────▼──────────────────────┐
│  MaxPooling2D(2×2)                      │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Flatten()                               │  ← 展平为一维向量
└──────────────────┬──────────────────────┘
┌──────────────────▼──────────────────────┐
│  Dropout(0.5)                            │  ← 随机丢弃50%节点，防过拟合
└──────────────────┬──────────────────────┘
┌──────────────────▼──────────────────────┐
│  Dense(64, ReLU)                         │  ← 全连接层，特征组合
└──────────────────┬──────────────────────┘
┌──────────────────▼──────────────────────┐
│  Dense(3, Softmax)                       │  ← 输出3类概率（总和=1）
└──────────────────┬──────────────────────┘
                   │
          ┌────────▼────────┐
          │  Output: 3 类   │
          │  paper / rock   │
          │  / scissors     │
          └─────────────────┘
```

**模型参数统计：** 约 200,000 个可训练参数

#### 2.3 编译模型参数

| 参数 | 配置 |
|------|------|
| **损失函数（Loss）** | sparse_categorical_crossentropy |
| **优化器（Optimizer）** | Adam |
| **性能指标（Metrics）** | accuracy（准确率） |
| **训练轮数（Epochs）** | 5 |

#### 2.4 训练过程

训练采用小批量梯度下降（Mini-batch SGD）方法：
1. 将训练数据打乱顺序
2. 按 Batch Size（64）分批输入模型
3. 每个 Batch 完成一次前向传播和反向传播更新参数
4. 每轮（Epoch）结束后在验证集上评估性能
5. 记录每轮的训练准确率、训练损失、验证准确率、验证损失

#### 3.4 详细分类报告

**生成文件：** `results/classification_report.txt`

**报告包含指标：**

| 指标 | 含义 | 理想值 |
|------|------|--------|
| **Precision（精确率）** | 预测为某类的样本中真正属于该类的比例 | 越高越好（接近 1.0） |
| **Recall（召回率）** | 真实为某类的样本中被正确预测的比例 | 越高越好（接近 1.0） |
| **F1-Score（F1分数）** | 精确率和召回率的调和平均值 | 越高越好（接近 1.0） |
| **Support（样本数）** | 该类别在测试集中的样本数量 | - |
| **Accuracy（准确率）** | 所有预测中正确预测的比例 | 越高越好（接近 1.0） |

## 五、实验结果分析

### 5.1 训练过程观察

典型的训练日志格式：

```
Epoch 1/5: loss=1.0234, accuracy=0.4850, val_loss=0.9856, val_accuracy=0.5417
Epoch 2/5: loss=0.7234, accuracy=0.6833, val_loss=0.8123, val_accuracy=0.6250
Epoch 3/5: loss=0.5123, accuracy=0.7917, val_loss=1.0234, val_accuracy=0.5833
Epoch 4/5: loss=0.3956, accuracy=0.8500, val_loss=1.2345, val_accuracy=0.5000
Epoch 5/5: loss=0.3362, accuracy=0.9467, val_loss=1.4951, val_accuracy=0.4625
```

**观察分析：**

| 现象 | 分析 |
|------|------|
| 训练损失持续下降 | ✅ 模型在训练集上学习有效 |
| 训练准确率持续上升 | ✅ 模型逐渐学会识别训练集中的特征 |
| 验证损失后期上升 | ⚠️ 可能存在过拟合现象 |
| 验证准确率低于训练准确率 | ⚠️ 模型泛化能力有待提升 |

### 5.2 过拟合现象分析

**什么是过拟合？**

过拟合是指模型在训练集上表现很好，但在测试集/验证集上表现较差的现象。就像一个学生只记住了考试题的答案，却没有真正理解知识点，考试遇到新题目就不会做了。

**本实验中的过拟合表现：**
1. 训练准确率（~94%）远高于验证准确率（~46%）
2. 训练损失持续下降，但验证损失后期上升
3. 模型可能过度记忆了训练集中的细节（如特定背景、光照）

**改进建议：**

| 方法 | 说明 | 操作示例 |
|------|------|----------|
| **数据增强** | 对训练图片进行随机变换，增加数据多样性 | 随机旋转 ±15°、水平翻转、亮度调整 |
| **Dropout** | 训练时随机丢弃部分神经元，防止过度依赖 | 将 Dropout(0.5) 调整为 Dropout(0.6) |
| **L2 正则化** | 在损失函数中添加权重惩罚，限制参数大小 | 添加 kernel_regularizer=l2(0.001) |
| **提前停止** | 监控验证损失，停止恶化时终止训练 | 使用 EarlyStopping(monitor='val_loss') |
| **更多数据** | 使用全部 2520 张训练图片，而非部分样本 | 将 max_train_per_class 从 200 改为 840 |

### 5.3 改进后的预期效果

通过以上改进措施，预期可以：
- ✅ 训练准确率略有下降（从 ~94% 降至 ~90%）
- ✅ 验证准确率显著提升（从 ~46% 提升至 ~80%+）
- ✅ 训练曲线和验证曲线趋于一致
- ✅ 模型泛化能力增强

### 9.3 使用训练好的模型进行推理

```python
import keras
import numpy as np
from PIL import Image

# 加载模型
model = keras.models.load_model('results/rps_model.keras')

# 准备待预测的图片
image_path = 'your_gesture_image.png'
img = Image.open(image_path).convert('RGB').resize((48, 48))

# 预处理
img_array = np.array(img, dtype=np.float32) / 255.0
img_array = np.transpose(img_array, (2, 0, 1))
img_array = np.expand_dims(img_array, axis=0)

# 进行预测
predictions = model.predict(img_array, verbose=0)
class_names = ['paper（布）', 'rock（石头）', 'scissors（剪刀）']

# 输出结果
predicted_index = np.argmax(predictions)
predicted_class = class_names[predicted_index]
confidence = predictions[0][predicted_index] * 100

print(f'预测结果: {predicted_class}')
print(f'置信度: {confidence:.1f}%')
print(f'各类别概率:')
for i, name in enumerate(class_names):
    print(f'  {name}: {predictions[0][i]*100:.1f}%')
```
